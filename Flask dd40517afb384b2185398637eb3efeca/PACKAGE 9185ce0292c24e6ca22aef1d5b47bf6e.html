Like we separate our form-oriented code to forms.py , we should move our models oriented code to models.py.

When we try to run like this, it will throw an error ‚Äúcircular import‚Äù.

This is because we are actually running flaskblog.py directly with python.

Anytime if python has to import something it will entirely run that module not any specific section.

Here, python tries to execute the models module but

when it comes to the second line, it returns to the flaskblog module, so while executing that module it again comes across from models import User which raises the circular import error

It fails to import db in models.py because it comes after to the line ‚Äúfrom models import User‚Äù.

When python directly runs the script it calls the name of the script ‚Äú__main__‚Äù. So, when executing the falskblog.py python know this script by the name ‚Äú__main__‚Äù.

When python gets to the ‚Äúfrom models import User‚Äù, it goes to the models module

Python is like ‚Äú I don‚Äôt know flashblog ( because it renamed that module as ‚Äú__main__‚Äù). So i go execute the module flaskblog

When it comes to the 4th line Python is like, ‚ÄúI know models module, but i haven‚Äôt seen User‚Äù. So, it throws the error ‚ÄúImportError: cannot import name 'User' from partially initialized module 'models‚Äô‚Äù.

The above is the reason why it fails to import User instead of failing to import db.

Pseudo soln:

When we execute flaskblog.py python goes to line 4 ‚Äúfrom models import User‚Äù. It will execute the models.py now on second line, Python says, ‚Äú Hey i already seen the module ‚Äò__main__‚Äô i know but i don‚Äôt see ‚Äòdb‚Äô there‚Äù. It gives the error ImportError: cannot import name 'db' from 'main' .

To resolve this,

We can move this line ‚Äúfrom models import User‚Äù down to db line. It will work.

This solution is ugly and still fails in situations like where the flaskblog.py was not set to ‚Äú__main__‚Äù. and we try to run the application. To make it clear look the at the example below,
